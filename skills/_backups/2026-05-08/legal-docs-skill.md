# Legal Docs — Template Generator (Not Legal Advice)

Generates template legal documents for web apps and SaaS targeting the Philippines market. Outputs are **templates**, not legal advice. Every generated document includes a mandatory disclaimer header and a TODO item to engage qualified counsel before publication.

This skill exists because solo founders need a starting point — not because Claude can practice law.

---

## Mandatory Safety Rules

These are non-negotiable:

1. **Every generated document must include the disclaimer header verbatim** (see below)
2. **Never claim templates are legally binding, valid, or sufficient in any jurisdiction**
3. **Always recommend qualified counsel review** before the user publishes any generated document
4. **Default to mild, user-friendly language** — avoid aggressive liability waivers, sweeping indemnities, or creative arbitration clauses
5. **Privacy Policy generation is OUT OF SCOPE** — delegate to `/ph-dpa-compliance` for PH-targeted projects (it handles consent + RA 10173 specifics)
6. **TODO writeback always includes "engage qualified counsel for review before publication"** as a Critical item

### Mandatory Disclaimer Header

Every generated document starts with this exact block:

```markdown
> **TEMPLATE — NOT LEGAL ADVICE**
>
> This document is a template generated by an AI assistant. It is not legal advice and is
> not a substitute for review by a qualified attorney licensed to practice in your
> jurisdiction. Laws vary by country, region, industry, and business model. Specific
> clauses (liability limits, dispute resolution, warranty disclaimers, IP terms) may be
> unenforceable in your jurisdiction without proper localization.
>
> Before publishing this document or relying on it: **engage qualified legal counsel.**
>
> Generated: YYYY-MM-DD | Last reviewed by counsel: [pending]
```

## Documents This Skill Generates

| Document | When to use |
|---|---|
| **Terms of Service** | Any product with user accounts or paid features |
| **Refund Policy** | Any product that accepts payments |
| **Cookie Policy** | Any site using cookies for analytics, ads, or session beyond essential |

| Document | Where to handle |
|---|---|
| **Privacy Policy** | Use `/ph-dpa-compliance` — covers consent, data subject rights, retention per RA 10173 |

| **Vendor / customer contracts** | Counsel — bilateral negotiated agreements |
| **Employment agreements / NDAs** | Counsel + HR |
| **Intellectual property assignments** | Counsel |
| **Anything for jurisdictions outside the Philippines** | Counsel in target jurisdiction |

Detect project context:

```bash
# CLAUDE.md / README for business model hints
grep -i "saas\|marketplace\|ecom\|subscription\|payment\|free\|paid" CLAUDE.md README.md 2>/dev/null

# Detect AI usage (triggers AI Output Disclaimer recommendation)
grep -rE "@anthropic-ai/sdk|openai|claude|gpt-" --include="package.json" --include="*.ts" -l 2>/dev/null | head -3

# Detect payment integrations (triggers Refund Policy)
grep -rE "stripe|paymongo|gcash|maya|paypal" --include="package.json" -l 2>/dev/null | head -3
```

If the project is not clearly PH-targeted, ask once:

Default: **Philippines**.

## Phase 2 — Context Gathering

Ask in **one block** (not iterative). Default unknowns to placeholders the user can fill later.

Required:
- Company / product name
- Business contact email (for legal notices)
- Country of operation (default: Philippines)
- Brief description of what the product does (1 sentence)

If Refund Policy:
- Refund window (default: 7 days)
- Eligible items (default: all paid items except digital goods that have been accessed)
- Refund method (default: original payment method, processed in 5–10 business days)

If SLA:
- Uptime target (default: 99.5% monthly for early-stage; 99.9% only if you actually have HA)
- Support response time (default: best effort; specify hours if you have a paid support tier)

If AI Output Disclaimer:
- Model(s) used (default: Claude Sonnet 4.6)
- What outputs are surfaced (default: text content)
- Whether outputs are reviewed by humans before display (default: no)

Never ask for: registered business address, registration number, tax ID — leave as `[COMPANY ADDRESS]`, `[SEC REGISTRATION NUMBER]` placeholders. The user will fill these in.
